# MongoDB Presentation Template — Slide Layout Catalog

This is the complete catalog of approved slide layouts from the MongoDB Presentation Template. Every presentation should use only these layouts. Each layout is available in dark (Slate bg) and light (White bg) variants unless noted otherwise.

## Layout Selection Guide

| Content Need | Recommended Layouts |
|-------------|-------------------|
| Opening a presentation | Welcome / Title |
| Setting expectations | Agenda |
| Starting a new section | Chapter Break |
| Making a key point | Big Bold Statement |
| Explaining a concept | Text (various), Photography + Text |
| Showing code | Code Slides |
| Listing features/benefits | Icon-Based (3 or 4 column) |
| Showing data | Charts, Tables, Timelines |
| Social proof | Quote, Case Study |
| Showing partners/tech | Partner Logos, Map |
| Showing product UI | Webpage Slides |
| Illustrating architecture | Illustration-Based, Diagram slides |
| Ending a presentation | Closing |

---

## 1. Welcome / Title Slides

The opening slide of every presentation.

### Layout: Welcome Dark
- **Background**: Slate (#001E2B) with Evergreen/Forest Green organic blobs
- **Elements**: MongoDB full logo (white), presentation title (Source Serif Pro, White, 48-54pt), subtitle/date (Source Code Pro, ALL CAPS, Spring Green or White)
- **Title**: 1-3 lines maximum
- **Subtitle**: 1 line, ALL CAPS

### Layout: Welcome Light
- **Background**: White with Mist/Lavender organic blobs
- **Elements**: MongoDB full logo (dark), presentation title (Source Serif Pro, Slate, 48-54pt), subtitle (Source Code Pro, ALL CAPS, Slate)

---

## 2. Agenda Slides

Used to show the presentation structure. Place after the title slide for presentations with 10+ slides.

### Layout: Agenda (Dark/Light)
- **Background**: Slate or White
- **Elements**: "Agenda" heading, numbered list of topics
- **Numbers**: Use Spring Green numbered circles from the Numbers/Nodes icon set
- **Topic text**: Lexend Deca Normal, 3-6 items maximum
- **Description**: Optional short description per item in Lexend Deca Light

---

## 3. Chapter Break Slides

Used to visually separate major sections of the presentation.

### Layout: Chapter Break Dark
- **Background**: Evergreen (#023430) with Forest Green organic blobs in top-right
- **Elements**: Section title (Source Serif Pro, White, 44-48pt), MongoDB leaf (Spring Green, top-right)
- **Title**: 1-2 lines, bottom-left of slide

### Layout: Chapter Break Light
- **Background**: White with Spring Green organic line and green blob in bottom-right
- **Elements**: Section title (Source Serif Pro, Slate, 44-48pt), MongoDB leaf (Slate, top-right)

---

## 4. Big Bold Statement Slides

For impactful single-message slides. The headline IS the content.

### Layout: Statement Centered (Dark/Light)
- **Background**: Slate or White
- **Elements**: Large headline centered on slide (Source Serif Pro, 44-54pt)
- **Max lines**: 2-4

### Layout: Statement with Category (Dark/Light)
- **Background**: Slate or White
- **Elements**: Category pill badge at top-center, large headline centered below
- **Headline**: 1-3 lines (Source Serif Pro, 40-48pt)
- **Subtitle**: Optional, 1 line small text at bottom

---

## 5. Text Slides

For explanatory content with headlines and body text.

### Layout: Text Left-Aligned (Dark/Light)
- **Elements**: Category pill (top-left), headline (1-2 lines, left), body text block (left, below headline)
- **Body**: 5-8 lines maximum, Lexend Deca Light
- **Leaf logo**: top-right

### Layout: Text with Subheadings (Dark/Light)
- **Elements**: 2-3 subheading + paragraph blocks arranged vertically on right half
- **Subheadings**: Lexend Deca Normal (bold), followed by body text
- **Use for**: Multiple related points on one slide

### Layout: Text Two-Column
- **Elements**: Headline spanning top, two equal columns of body text below
- **Per column**: 5-7 lines maximum

### Layout: Numbered Bullets (Dark/Light)
- **Elements**: 3 numbered items with Spring Green number circles, each with a heading and 1-2 lines of body
- **Numbers**: Use the approved numbered circle icons (1, 2, 3)

---

## 6. Code Slides

For showing code examples. Always use dark background for code blocks.

### Layout: Code Full-Width Dark
- **Background**: Slate (#001E2B)
- **Elements**: Category pill (top), headline (Source Serif Pro, White), code block below
- **Code**: Source Code Pro Medium, 12-14pt, White text
- **Max lines**: ~15 lines of code
- **Syntax highlighting**: Use Spring Green for keywords/important tokens

### Layout: Code Left + Text Right
- **Elements**: Code block on left half (dark bg), explanatory text on right half
- **Code block**: Rounded-corner dark container
- **Text**: Headline + body explaining the code

### Layout: Code Right + Text Left
- Same as above but mirrored — text left, code right

### Layout: Code with Output
- **Elements**: Code block on top/left, output/result on bottom/right
- **Output area**: Lighter shade or different container to distinguish from code

---

## 7. Quote Slides

For customer testimonials and notable quotes.

### Layout: Quote Full (Dark)
- **Background**: Forest Green (#00684A) with organic blobs
- **Elements**: Large opening quotation mark (Spring Green), quote text (Source Serif Pro, White, 28-36pt), attribution line (Source Code Pro, ALL CAPS, Spring Green)
- **Quote**: 3-5 lines maximum
- **Attribution**: "FIRSTNAME LASTNAME | COMPANY" format

### Layout: Quote Full (Light)
- **Background**: White
- **Elements**: Same structure, Slate text, Forest Green quotation mark

### Layout: Quote with Photography
- **Elements**: Photo on left half (in container with organic shapes), quote + attribution on right half
- **Available in**: Dark (Forest Green bg) and light (White bg)

---

## 8. Icon-Based Slides

For showing features, benefits, or capabilities using MongoDB icons.

### Layout: Three Icons (Dark/Light)
- **Elements**: Category pill, headline, 3 icons in a row with captions below
- **Icon size**: 0.96" maximum
- **Caption**: Lexend Deca, 2 lines maximum per caption

### Layout: Four Icons (Dark/Light)
- **Elements**: Category pill, headline, 4 icons in a row with captions
- Same rules as 3-icon but icons slightly smaller to fit

### Layout: Icons with Descriptions
- **Elements**: 3 or 4 icons, each with a bold heading + 2-3 lines of description below
- **Use for**: Feature comparisons, capability overviews

---

## 9. Illustration-Based Slides

Use MongoDB's approved spot illustrations (database, document, data layers).

### Layout: Illustration Left + Text Right (Dark/Light)
- **Elements**: Spot illustration on left, headline + body text on right
- **Illustrations available**: Database cylinder, document page, stacked data layers

### Layout: Illustration Right + Text Left (Dark/Light)
- Same but mirrored

### Layout: Three Illustrations
- **Elements**: 3 illustrations in a row, each with heading + description below
- **Use for**: Comparing database/document/data-layer concepts

---

## 10. Photography-Based Slides

For slides combining photography with text content.

### Layout: Photo Left + Text Right (Dark/Light)
- **Photo**: Occupies left ~40-50% of slide, in rounded-corner container
- **Text side**: Category pill, headline (1-2 lines), body (5 lines max)
- **Dark variant**: Slate bg on text side
- **Light variant**: White bg on text side

### Layout: Photo Right + Text Left (Dark/Light)
- Same but mirrored

### Layout: Photo Left + Title Only (Light)
- **Elements**: Photo on left half, large headline on right (up to 4 lines), no body text
- **Use for**: Bold visual statements

### Layout: Photo with Subheadings
- **Elements**: Photo on left, 3 subheading + description blocks on right
- **Per block**: Bold heading + 2-3 lines body

### Layout: Photo with Checkmarks
- **Elements**: Photo on left half, 3 checkmark items on right
- **Checkmarks**: Spring Green badge-style checkmarks
- **Per item**: 2 lines of text maximum
- **Dark and light variants available**

### Layout: Photo with Organic Shapes (Dark)
- **Background**: Evergreen or Forest Green
- **Elements**: Photo in rounded container with Spring Green outline shapes overlaid, text on opposite side
- **Use for**: More visually dynamic slides

### Layout: Photo Full-Bleed + Text Overlay
- **Elements**: Photo occupies right half edge-to-edge, headline on left with body text
- **Background of text area**: Spring Green block behind the photo subject
- **Headline**: 1-2 lines

---

## 11. Image-Based Slides

For smaller images (not full photography) combined with text.

### Layout: Two Images Left + Text Right
- **Elements**: Two small images stacked on left (in rounded containers with Spring Green shapes), category pill + headline + body on right
- **Background**: Light (White) or dark (Slate)

### Layout: Image + Text (Dark with shapes)
- **Elements**: Small image in organic shape container, category pill, headline, body text
- **Background**: Evergreen with blob shapes

---

## 12. Webpage / Screenshot Slides

For showing product UI, dashboards, or website screenshots.

### Layout: Screenshot Centered (Dark)
- **Background**: Slate
- **Elements**: Title at top (Source Serif Pro, White), large screenshot below in browser-frame mockup
- **Screenshot**: Realistic browser chrome with dots (red/yellow/green) at top

### Layout: Screenshot + Text (Dark/Light)
- **Elements**: Screenshot on right half (in browser frame), headline + body text on left
- **Headline**: 1-2 lines
- **Body**: 3-5 lines

---

## 13. Case Study Slides

For showcasing customer success stories.

### Layout: Case Study
- **Elements**: Customer logo (top-left, in light rounded container), 3 statistics with labels (left column), 3 bullet point descriptions (right column)
- **Statistics**: Large number in Forest Green (Source Serif Pro, 36pt), ALL CAPS label above (Source Code Pro)
- **Descriptions**: Bold heading + 1-2 lines body each
- **Variant with 2 logos**: For joint case studies

---

## 14. Timeline Slides

For showing progression, history, or roadmap.

### Layout: Three-Point Timeline (Dark/Light)
- **Elements**: Category pill + title at top, horizontal dashed line with 3 Spring Green dots, 3 columns below with heading + body
- **Per column**: Bold heading, 3 lines body

### Layout: Six-Point Timeline (Dark/Light)
- Same as above but 6 points on the timeline
- **Per column**: Bold heading, 5-6 lines body (smaller text)

### Layout: Timeline with Date Badges
- **Elements**: Title at top, Spring Green pill badges with dates/years above the timeline dots
- **Light and dark variants**
- **Use for**: Product history, project milestones

---

## 15. Charts and Graphs

### Layout: Simple Table (Light)
- **Elements**: Chart title + description on left, 3-column table on right
- **Table**: Clean lines, header row in Slate text, up to 5 data rows
- **Header text**: Italicized, Forest Green

### Layout: Styled Table (Dark)
- **Elements**: Category pill + title at top, full-width table
- **Header row**: Spring Green background, Slate text
- **Data rows**: White text on Slate, alternating subtle borders
- **Max**: ~6 rows, 3 columns

### Layout: Comparison / Feature Matrix
- **Elements**: Full-width table with Spring Green header row, checkmark symbols in cells
- **Highlight row**: Lavender background for emphasis
- **Max**: 12 rows, 8 columns
- **Use for**: Feature comparisons, product tiers

### Layout: Vertical Bar Chart (Dark/Light)
- **Elements**: Headline on left, 4 ascending bars on right
- **Bar colors**: Use Lavender (lowest) progressing to Spring Green (highest) — or use chart-only colors for multi-series
- **Labels**: Percentage above each bar, ALL CAPS category below
- **Do not** add or remove bars from the template — use slide 137+ for more bars

### Layout: Horizontal Bar Chart (Dark/Light)
- **Elements**: Headline on left, 3 horizontal bars stacked vertically
- **Colors**: Spring Green (largest), Lavender, Mist — or chart-only colors
- **Labels**: Topic label left of bar, value inside bar

### Layout: Line Chart (Dark)
- **Elements**: Category pill + title, line chart with 2 data series
- **Line colors**: Chart-only colors (Lime, Sky)
- **Grid**: Dashed horizontal lines, axis labels

### Layout: Grouped Bar Chart (Dark)
- **Elements**: Category pill + title, grouped vertical bars with 3 data series
- **Colors**: 3 chart-only colors for 3 series
- **Legend**: Color squares with labels in top-right

### Layout: Donut Chart (Dark)
- **Elements**: Category pill, one-word headline, body text on left; donut chart on right
- **Variants**: 2-value, 4-value, 6-value donuts
- **Colors**: Chart-only colors
- **Percentages**: Displayed inside/beside each segment

### Layout: Bubble Chart (Dark)
- **Elements**: X/Y grid, Spring Green circles of varying sizes positioned on grid
- **Labels**: Project/item names inside circles
- **Axes**: Labeled with "VALUE" text

### Layout: 3x3 Grid Matrix (Dark)
- **Elements**: 9 white cards in 3x3 grid on Slate background
- **Per card**: 3 lines of text
- **Axes labels**: Values on left and bottom

### Layout: Spider / Radial Diagram (Dark)
- **Elements**: Central Spring Green circle connected to 12 surrounding smaller circles
- **Surrounding circles**: Chart-only colors (Sky, Azure, Lime, etc.)
- **Labels**: Inside each circle

### Layout: Process Diagrams (Dark)
- **Circular (4-step)**: 4 interlocking circular arrows with numbers, label + description below each
- **Triangle (3-point)**: Numbered triangle with 3 descriptions
- **Diamond (4-point)**: Numbered diamond shape with 4 descriptions
- **Pentagon (5-point)**: Numbered pentagon with 5 descriptions
- **Hexagon (6-point)**: Numbered hexagon with 6 descriptions
- **Colors**: Chart-only colors for each segment
- **Title**: Single line at top (Source Serif Pro)

## 16. Partner and Language Logo Slides

### Layout: Three Cloud Provider Logos
- **Elements**: Category pill + title, 3 logo cards (AWS, Google Cloud, Azure) on world map background
- **Logos**: On white rounded-rectangle cards
- **Dark and light variants**

### Layout: Six Logo Grid (Light)
- **Elements**: Category pill + title, 3x2 grid of logos
- **Use for**: Customer logos, technology partners

### Layout: Nine Logo Grid (Light)
- **Elements**: Title, 3x3 grid of logos
- **Logos**: Full color when on white background, single color when >6 logos

### Layout: Logo + Text (Dark/Light)
- **Elements**: 3 single-color logos stacked vertically on left, title + 3 description blocks on right
- **When showing logos with text**: Always use single-color logos
- **Multiple logos should always be single color** — reduces visual clutter, projects professionalism

### Layout: World Map (Dark)
- **Elements**: Category pill + title, world map graphic in Forest Green on Slate background
- **Optional**: Cloud provider logos overlaid

## 17. Closing Slides

The final slide of every presentation.

### Layout: Closing Light
- **Background**: White with Lavender blob (bottom-left) and Spring Green curved line (top-right)
- **Text**: "Thank you for your time." centered (Source Serif Pro, Slate, 44pt)
- **Leaf logo**: top-right (Slate)

### Layout: Closing Dark
- **Background**: Slate (#001E2B) with Forest Green blob (bottom-left) and Spring Green curved line
- **Text**: "Thank you for your time." centered (Source Serif Pro, White, 44pt)
- **Leaf logo**: top-right (Spring Green)

## Content Limits Quick Reference

| Layout Type | Max Headline Lines | Max Body Lines | Max Items |
|------------|-------------------|----------------|-----------|
| Title / Welcome | 3 | - | - |
| Chapter break | 2 | - | - |
| Big bold statement | 4 | - | - |
| Text with body | 2-3 | 5-8 | - |
| Code | 1-2 | ~15 code lines | - |
| Quote | - | 5 quote lines | 1 |
| Icon-based | 1 | 2 per caption | 3-4 icons |
| Photo + text | 2-3 | 5 | - |
| Photo + subheadings | - | 3 per block | 3 blocks |
| Case study | - | 2 per item | 3 stats + 3 bullets |
| Timeline 3-point | 2 | 3 per column | 3 points |
| Timeline 6-point | 2 | 5-6 per column | 6 points |
| Table | 1 | - | 5-6 rows, 3 cols |
| Bar chart | 1 | - | 4 bars |
| Partner logos | 1 | - | 3-9 logos |
| Closing | 1 | - | - |
